About

Manual-first. Detail-obsessed.

I'm a self-taught offensive security researcher focused on web application penetration testing. My background is in nursing — a field that trained precision, structured documentation, and analytical thinking under pressure. I carried those habits into security and built from there.

My approach is logic-driven before it's tool-driven. I map how an application behaves, trace data flows, and understand the architecture before looking for weaknesses. The issues that matter most — business logic flaws, trust boundary violations, multi-step auth chains — are exactly what automated scanners miss. That's where I focus.

I build tools to extend my analysis capabilities, not to replace thinking. Every tool I've published started as a specific gap I needed to fill during research.

Manual before automated

I start with clean requests and observe behavior. Tooling comes in after I understand the system, not before.

Architecture-level thinking

Real vulnerabilities live in how systems connect — not just in individual endpoints. I trace data flows end to end.

Builder, not just a user

Eight open-source tools: static analysis, recon, fuzzing, HTTP testing. Each built to close a real research gap.

Written output as standard

Two technical books published. Lab writeups formatted as professional reports — steps, impact, remediation.

Projects

Security Tools

Eight open-source tools built across Python, Go, and Bash — each addressing a specific gap in manual security research and penetration testing workflows.

TaintHound
Python

AST-based static taint analysis for client-side JavaScript. Traces data from user-controllable sources to dangerous sinks — surfaces DOM XSS, open redirects, and unsafe eval/Function usage. Uses tree-sitter, not regex. Two collection modes: local scan or headless Chromium crawl.

Static Analysis tree-sitter Taint Tracing CLI
Lurak
Bash

Modular Linux-first recon framework for attack surface enumeration. 14 independent modules — ASN recon, CIDR discovery, port scanning, subdomain enumeration (7 passive sources), DNS, HTTP fingerprinting. TUI interface, structured JSON output, Smart Pipeline for unattended automation.

Recon TUI 14 Modules Pipeline
AuthFlowVisualizer
Go + Node.js

Static analysis tool that uses the Babel AST to extract auth-related behavior from JavaScript applications, build call graphs, and generate CLI and HTML flow reports. Detects 20+ authentication keywords. Go backend with Node.js Babel bridge.

Babel AST Call Graphs Auth Analysis
Paramancer
Go

Concurrent HTTP parameter mutation engine for web app security testing. Probes every parameter using systematic mutations, then diffs each response against a clean baseline to surface anomalies. No scanner, no plugin — structured noise with a purpose.

Fuzzing Concurrent Baseline Diffing
Freecollapse
Python

Fuzzing tool for generating boundary-case payloads targeting normalization vulnerabilities. Produces mutation sets for path collapsing, Unicode handling, double-encoding, and null-byte injection. Inspired by reCollapse — extended for arbitrary input and full byte-range control.

Fuzzing Normalization Encoding
HttpRequestLab
Python / PySide6

Lightweight desktop HTTP requester for manual security testing. Raw HTTP editing, auto Host sync, follow-redirects toggle, full response viewer — a standalone Burp Repeater companion for rapid request crafting and inspection without the Burp overhead.

Desktop PySide6 HTTP Testing
HT-Detective
Go

Concurrent HTTP reconnaissance tool for attack surface profiling. Detects server technologies, CMS platforms, frameworks, and security header configuration. Structured JSON output designed for pipeline integration in pre-engagement recon workflows.

Recon Fingerprinting Concurrent
Watchtower
Python / Flask

Bug bounty automation platform. Subdomain enumeration, DNS resolution, HTTP probing, and Nuclei scanning across multiple programs — orchestrated via a modular Flask backend with MongoDB and configurable per-target pipelines.

Automation Flask Bug Bounty
